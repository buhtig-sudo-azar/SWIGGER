 1. НАЗНАЧЕНИЕ
═══════════════════════════════════════════════════════════════

WCD Gun — интерактивный инструмент для автоматизированного обнаружения
и эксплуатации уязвимости Web Cache Deception (WCD). 
Работает как в ручном, так и в связке с WCD Launcher (wcd_framework.sh).

Основные возможности:
  - Одиночные выстрелы (single / combo)
  - Массовый перебор (burst / fire)
  - Умный детектор кэша (6 уровней уверенности)
  - Сравнение тел ответов на утечку чувствительных данных
  - Подтверждение статуса кэша повторными запросами
  - Поддержка проксирования (Burp Suite)
  - Логирование успешных векторов
  - Управление обоймами (разделители, расширения, static dirs, эндпоинты)

═══════════════════════════════════════════════════════════════
 2. БЫСТРЫЙ СТАРТ
═══════════════════════════════════════════════════════════════

   chmod +x wcd_gun.sh
   ./wcd_gun.sh
   
При первом запуске скрипт создаст каталоги magazines/ и reports/,
а также дефолтные обоймы. Вы сразу увидите баннер, текущую конфигурацию
и шпаргалку команд.

Для немедленной атаки на готовый стенд:

   wcd> set target http://127.0.0.1:8080
   wcd> set cookie "session=abc123"
   wcd> profile          ← профилирование кэша
   wcd> burst            ← тотальный перебор

Или запустить с целью сразу (работает из лаунчера):

   ./wcd_gun.sh http://127.0.0.1:8080/profile

═══════════════════════════════════════════════════════════════
 3. КОМАНДЫ
═══════════════════════════════════════════════════════════════

── НАСТРОЙКА ─────────────────────────────────────────────────

  start               — мастер первоначальной настройки обойм
  set target <URL>    — задать цель (http://host:port)
  set cookie <строка> — установить cookie для прогрева кэша
  proxy on|off|status — включить/отключить проксирование через Burp
                        (по умолчанию 127.0.0.1:8082)

── РАЗВЕДКА ──────────────────────────────────────────────────

  target              — проверить доступность цели, показать кэш-заголовки
  cookie              — проверить валидность куки
  profile [URL]        — профилирование кэша (два запроса: с кукой и без,
                        сравнение тел ответов)

── АТАКА ─────────────────────────────────────────────────────

  single <путь>       — одиночный выстрел.
                        Пример: single /profile%3b.css
                        Можно и полный URL: single http://target/foo%3b.css
                        Делает два запроса: прогрев (с кукой) и проверку (без куки).
                        Выводит кэш-статус и сравнение тел.

  combo <sdir> <d> <e> <ep>
                       — ручная сборка вектора из компонентов.
                        sdir = static directory (или "none" если без неё)
                        d    = разделитель (%3B, ;, #, ?)
                        e    = расширение (.css, .js, .png)
                        ep   = endpoint (profile, account, admin)
                        Пример: combo /images %3B .css profile
                        Соберёт путь: /images/../profile%3B.css

  burst               — полный перебор всех комбинаций обойм.
                        Перемножает 4 обоймы (static dirs × разделители ×
                        расширения × эндпоинты). Может быть долгим.

  fire                — быстрый перебор (только разделители × расширения).
                        Полезен для первичной разведки.

── УПРАВЛЕНИЕ ОБОЙМАМИ ────────────────────────────────────────

  load <тип> <имя>    — загрузить обойму.
                        Типы: delimiters, extensions, staticdir, endpoints
                        Пример: load delimiters encoded
                        Загрузит magazines/encoded.delimiters

  add <разделитель>   — добавить разделитель в текущую обойму "на лету".
                        Пример: add %0A

  show                — показать текущее состояние: цель, прокси, обоймы.

── ПРОЧЕЕ ────────────────────────────────────────────────────

  help                — полная справка
  quit / exit         — выход

 4. ДЕТЕКТОР КЭША (6 уровней уверенности)
═══════════════════════════════════════════════════════════════

После каждого запроса скрипт анализирует заголовки ответа:

  HIT_STRONG  [✓✓]  — явный HIT + возраст >0. Уверенность 95%.
  HIT_WEAK    [✓]   — HIT без возраста, либо возраст без HIT. 40-65%.
  MISS_STRONG [✗]   — явный MISS. Уверенность 90%.
  MISS_WEAK   [?]   — есть кэш-заголовки, но статус не ясен. 30%.
  DYNAMIC     [~]   — no-cache, private, must-revalidate. 90%.
  UNKNOWN     [ ]   — нет никаких кэш-заголовков.

Успешной атакой считается HIT_STRONG или HIT_WEAK.

═══════════════════════════════════════════════════════════════
 5. СРАВНЕНИЕ ТЕЛ ОТВЕТОВ
═══════════════════════════════════════════════════════════════

Для single и combo скрипт сравнивает тело ответа "прогрева" (с кукой)
и тело ответа "проверки" (без куки). Оцениваются:

  - Разница в длине (в байтах и процентах).
    Если разница ≤ 10% — длины СОВПАДАЮТ.
  - Наличие чувствительных маркеров (email, account, token,
    session, password, admin, api, secret, credit, card и др.).

При совпадении длин и наличии маркеров выводится:
"ВЕРОЯТНАЯ УТЕЧКА!"

═══════════════════════════════════════════════════════════════
 6. ОБОЙМЫ (MAGAZINES)
═══════════════════════════════════════════════════════════════

Обоймы — это текстовые файлы в каталоге magazines/.
Каждая строка — один элемент (игнорируются пустые строки и # комментарии).

Типы обойм:

  *.delimiters  — разделители:  ;  %3B  ?  #  %23  %3F  %0A  ...
  *.extensions  — расширения:   .css  .js  .jpg  .png  .svg  ...
  *.staticdir   — static dirs:  /static  /images  /assets  /js  ...
  *.endpoints   — эндпоинты:    profile  account  admin  dashboard  ...

Дефолтные обоймы создаются при первом запуске:

  default.delimiters   — базовый набор разделителей
  default.extensions   — популярные статические расширения
  default.staticdir    — типовые пути к статике
  default.endpoints    — популярные эндпоинты
  encoded.delimiters   — только URL-encoded разделители

Создавай свои обоймы для конкретных целей, например:

  magazines/target_x.delimiters
  magazines/target_x.extensions

И загружай: load delimiters target_x

═══════════════════════════════════════════════════════════════
 7. ИНТЕГРАЦИЯ С WCD LAUNCHER
═══════════════════════════════════════════════════════════════

WCD Launcher (wcd_framework.sh) поднимает уязвимый стенд в RAM
и может запустить пулемёт командой "gun". Лаунчер передаёт пулемёту
цель как аргумент, например:

   ./wcd_gun.sh http://127.0.0.1:8080/profile

Пулемёт автоматически устанавливает TARGET_HOST и переходит
в интерактивный режим.

═══════════════════════════════════════════════════════════════
 8. ПРОКСИРОВАНИЕ (Burp Suite)
═══════════════════════════════════════════════════════════════

Для анализа трафика через Burp:

   wcd> proxy on          ← включить (Burp должен быть запущен)
   wcd> proxy off         ← выключить
   wcd> proxy status      ← текущий статус

Прокси по умолчанию: 127.0.0.1:8082
Изменить: export BURP_HOST=... BURP_PORT=... перед запуском.

10. ПРИМЕРЫ ПОЛНОГО ЦИКЛА
═══════════════════════════════════════════════════════════════

Пример 1: Быстрая атака на один вектор

   wcd> set target http://127.0.0.1:8080
   wcd> set cookie "session=eyJ1c2VyIjoiYW5hdG9seSJ9"
   wcd> combo /images %3B .css profile
   
   (Если HIT — уязвимость подтверждена)

Пример 2: Разведка незнакомого хоста

   wcd> set target https://target.com
   wcd> profile
   wcd> fire
   
   (Ищем, какие разделители и расширения кэшируются)

Пример 3: Глубокий перебор со своими обоймами

   wcd> load delimiters encoded
   wcd> load staticdir my_static_paths
   wcd> burst
   
   (Перебирает все комбинации с загруженными обоймами)

11. УСТРАНЕНИЕ НЕИСПРАВНОСТЕЙ
═══════════════════════════════════════════════════════════════

Проблема: "Нет ответа от ..."
  → цель не запущена, неверный порт, или сеть недоступна.
  → проверь: target

Проблема: Пулемёт молча вылетает при запуске
  → убедись, что версия 10.4 (в баннере),
  → проверь права на выполнение (chmod +x),
  → запусти с VERBOSE=true для деталей.

Проблема: Все запросы возвращают MISS
  → возможно, цель не кэширует. Проверь: profile
  → стенд в режиме "secure" - переключи на "vuln" в лаунчере.
  → или кэш уже протух — увеличь время между прогревом и проверкой.
