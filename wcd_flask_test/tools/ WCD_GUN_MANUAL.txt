 1. НАЗНАЧЕНИЕ
═══════════════════════════════════════════════════════════════

WCD Gun — интерактивный инструмент для автоматизированного обнаружения
и эксплуатации уязвимости Web Cache Deception (WCD). 
Работает как в ручном режиме, так и в связке с WCD Launcher.

Основные возможности v10.5:
  - Одиночные выстрелы (single / combo)
  - Массовый перебор (burst / fire) с прерыванием по Ctrl+C
  - Умный детектор кэша (6 уровней уверенности)
  - Сравнение тел ответов на утечку чувствительных данных
  - Подтверждение статуса кэша повторными запросами
  - Поддержка проксирования (Burp Suite)
  - Просмотр обойм (magazines) и хитов (hits) без выхода
  - Просмотр активных процессов (ps)
  - Логирование успешных векторов
  - Управление обоймами (разделители, расширения, static dirs, эндпоинты)

═══════════════════════════════════════════════════════════════
 2. БЫСТРЫЙ СТАРТ
═══════════════════════════════════════════════════════════════

   chmod +x wcd_gun.sh
   ./wcd_gun.sh
   
При первом запуске создаются каталоги magazines/ и reports/,
а также дефолтные обоймы. Сразу виден баннер, конфигурация,
активные процессы и шпаргалка команд.

Для атаки на готовый стенд:

   wcd> set target http://127.0.0.1:8080
   wcd> set cookie "session=abc123"
   wcd> profile          ← профилирование кэша
   wcd> burst            ← тотальный перебор

Или из лаунчера:

   wcd-launcher > gun

Пулемёт получит цель автоматически.

═══════════════════════════════════════════════════════════════
 3. КОМАНДЫ
═══════════════════════════════════════════════════════════════

── НАСТРОЙКА ─────────────────────────────────────────────────

  start               — мастер первоначальной настройки обойм
  set target <URL>    — задать цель (http://host:port)
  set cookie <строка> — установить cookie для прогрева кэша
  proxy on|off|status — проксирование через Burp (127.0.0.1:8082)

── РАЗВЕДКА ──────────────────────────────────────────────────

  target              — проверить доступность цели, показать кэш-заголовки
  cookie              — проверить валидность куки
  profile [URL]       — профилирование кэша (с кукой / без куки, сравнение тел)

── АТАКА ─────────────────────────────────────────────────────

  single <путь>       — одиночный выстрел.
                        Прогрев с кукой → проверка без куки.
                        Пример: single /profile%3b.css
                        Можно полный URL: single http://target/foo%3b.css

  combo <sdir> <d> <e> <ep>
                       — ручная сборка вектора.
                        sdir = static directory (none — без неё)
                        d    = разделитель (;, %3B, ?, #)
                        e    = расширение (.css, %2ecss, .js)
                        ep   = endpoint (profile, api%2fkeys)
                        Пример: combo /images %3B .css profile
                        → /images/../profile%3B.css

  burst               — полный перебор (static dirs × разделители ×
                        расширения × эндпоинты). Можно прервать Ctrl+C.

  fire                — быстрый перебор (разделители × расширения).
                        Можно прервать Ctrl+C.

── УПРАВЛЕНИЕ ОБОЙМАМИ ────────────────────────────────────────

  load <тип> <имя>    — загрузить обойму.
                        Типы: delimiters, extensions, staticdir, endpoints
                        Пример: load delimiters normalization

  add <разделитель>   — добавить разделитель в текущую обойму.
                        Пример: add %0A

  magazines           — показать все обоймы в магазине

── ИНФОРМАЦИЯ ────────────────────────────────────────────────

  show                — показать цель, прокси, текущие обоймы
  hits                — показать успешные хиты (из successful_vectors.log)
  ps                  — показать активные процессы (wcd, flask, nginx)
  help                — полная справка
  quit / exit         — выход

═══════════════════════════════════════════════════════════════
 4. ДЕТЕКТОР КЭША (6 уровней уверенности)
═══════════════════════════════════════════════════════════════

  HIT_STRONG  [✓✓]  — явный HIT + Age > 0. Уверенность 95%.
  HIT_WEAK    [✓]   — HIT без Age, либо Age без HIT. 40-65%.
  MISS_STRONG [✗]   — явный MISS. Уверенность 90%.
  MISS_WEAK   [?]   — есть кэш-заголовки, статус не ясен. 30%.
  DYNAMIC     [~]   — Cache-Control: no-cache/private/must-revalidate. 90%.
  UNKNOWN     [ ]   — нет кэш-заголовков.

Успех = HIT_STRONG или HIT_WEAK.

═══════════════════════════════════════════════════════════════
 5. СРАВНЕНИЕ ТЕЛ ОТВЕТОВ
═══════════════════════════════════════════════════════════════

Для single/combo сравниваются тело "прогрева" (с кукой) и "проверки" (без):

  - Разница в длине ≤ 10% → длины СОВПАДАЮТ
  - Наличие маркеров: email, @, profile, account, dashboard, admin,
    token, session, api, secret, password, credit, card

Совпадение длин + маркеры = "ВЕРОЯТНАЯ УТЕЧКА!"

═══════════════════════════════════════════════════════════════
 6. ОБОЙМЫ (MAGAZINES)
═══════════════════════════════════════════════════════════════

Файлы в magazines/. Строки — элементы. Пустые и #comment игнорируются.

Типы:
  *.delimiters  — ;  %3B  ?  #  %23  %3F  %0A
  *.extensions  — .css  .js  .jpg  %2ecss  %2ejs
  *.staticdir   — /static  /images  /assets  /static/..%2f
  *.endpoints   — profile  account  admin  api%2fkeys

Встроенные обоймы:
  default.*        — базовые наборы
  encoded.delimiters — только URL-encoded разделители

Свои обоймы:
  normalization.*  — для Normalization Discrepancy (создаются вручную)

═══════════════════════════════════════════════════════════════
 7. NORMALIZATION DISCREPANCY (новая техника)
═══════════════════════════════════════════════════════════════

Расхождение нормализации между кэшем и бэкендом.

Суть: кэш видит /static/..%2fprofile%3B.css как static-файл (префикс /static),
бэкенд декодирует %2f → /, обрабатывает ../ → /profile;.css — отдаёт профиль.

Обоймы normalization.* содержат:
  staticdir:  /static/..%2f, /images/..%252f, ..%2f (без static dir)
  delimiters: ;, %3B, ?, %3F
  extensions: .css, %2ecss, .js, %2ejs
  endpoints:  profile, account, api%2fkeys

Загрузка:
   wcd> load staticdir normalization
   wcd> load delimiters normalization
   wcd> load extensions normalization
   wcd> load endpoints normalization
   wcd> burst

═══════════════════════════════════════════════════════════════
 8. ИНТЕГРАЦИЯ С WCD LAUNCHER
═══════════════════════════════════════════════════════════════

Лаунчер (wcd_framework.sh) поднимает стенд Flask + Nginx в RAM.
Команда "gun" запускает пулемёт с целью http://127.0.0.1:8080/profile.

Ctrl+C в пулемёте → возврат в меню лаунчера, стенд остаётся жив.

═══════════════════════════════════════════════════════════════
 9. CTRL+C — ПРЕРЫВАНИЕ ОПЕРАЦИЙ
═══════════════════════════════════════════════════════════════

  - В меню: не выходит, пишет "Используй quit для выхода"
  - Во время burst/fire: останавливает перебор, возвращает в меню
  - Хиты, найденные до прерывания, сохраняются в successful_vectors.log
  - Команда hits покажет всё что найдено

═══════════════════════════════════════════════════════════════
10. ПРОКСИРОВАНИЕ (Burp Suite)
═══════════════════════════════════════════════════════════════

   wcd> proxy on          — включить (Burp должен быть запущен)
   wcd> proxy off         — выключить
   wcd> proxy status      — текущий статус

По умолчанию: 127.0.0.1:8082
Изменить: export BURP_HOST=... BURP_PORT=...

13. УСТРАНЕНИЕ НЕИСПРАВНОСТЕЙ
═══════════════════════════════════════════════════════════════

"Нет ответа от ..."
  → проверь цель: target
  → стенд запущен? ps

Пулемёт вылетает при запуске
  → проверь версию в баннере (должна быть 10.5)
  → chmod +x wcd_gun.sh
  → VERBOSE=true ./wcd_gun.sh

Все ответы MISS
  → проверь режим стенда: в лаунчере должен быть vuln
  → profile — посмотри кэш-заголовки
  → кэш протух — увеличь sleep в single_shot

Ctrl+C не останавливает burst
  → обнови до v10.5 (добавлен флаг INTERRUPTED)
