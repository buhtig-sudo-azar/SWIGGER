                    WCD GUN v7.7 — РУКОВОДСТВО ПОЛЬЗОВАТЕЛЯ
                         Web Cache Deception Пулемёт

ОПИСАНИЕ
--------
wcd_gun.sh — это интерактивный инструмент для автоматизированного тестирования
уязвимости Web Cache Deception (WCD). Он имитирует двухфазную атаку:
  1. Прогрев кэша (запросы с кукой аутентификации от имени жертвы)
  2. Проверка кэша (запросы без куки от имени атакующего)

Инструмент поддерживает гибкое управление разделителями и расширениями через
"обоймы", интеграцию с Burp Suite для анализа трафика и детальное логирование.

ЗАПУСК
------
./wcd_gun.sh [URL_цели]

Если URL передан как аргумент, пулемёт запускается в автоматическом режиме:
  1. Запрашивает куку
  2. Предлагает включить проксирование через Burp Suite
  3. Сразу начинает полный обстрел (fire)

Без аргументов запускается интерактивный режим с командной строкой.

ИНТЕРАКТИВНЫЙ РЕЖИМ
--------------------
После запуска вы попадаете в оболочку с приглашением "wcd-gun > ".
Основные категории команд:

🎯 ЦЕЛЬ И КУКА
--------------
set target <URL>      - установить URL атакуемой цели
set cookie <COOKIE>   - установить куку аутентификации (например, "session=abc123")
target                - показать текущую цель и проверить её доступность
cookie                - показать текущую куку и проверить её валидность

Пример:
  wcd-gun > set target http://127.0.0.1:8080/profile
  wcd-gun > set cookie session=valid

🔌 ПРОКСИ (Burp Suite)
----------------------
proxy on              - включить проксирование через Burp Suite (127.0.0.1:8082)
proxy off             - отключить проксирование
proxy                 - показать текущий статус прокси

При включении скрипт проверяет доступность Burp Suite. Если Burp не запущен,
предлагает запустить его автоматически (работает для Kali Linux, стандартных
установок). Весь трафик curl пойдёт через Burp, и вы сможете видеть запросы
в HTTP History.

🔫 ОДИНОЧНЫЕ ВЫСТРЕЛЫ
---------------------
single [разделитель] [расширение]
    Полный цикл: прогрев кэша с кукой (если задана) → пауза 2 сек →
    проверка без куки. Отображает результат с анализом заголовков.

quick [разделитель] [расширение]
    Быстрая проверка: только один запрос без куки, показывает статус кэша.

show [разделитель] [расширение]
    Показать полный HTTP-ответ (заголовки + тело) для отладки.

Параметры можно опускать. Если параметр не указан, берётся первый элемент
из текущей обоймы (разделителей или расширений).
Специальное значение "none" отключает параметр (разделитель или расширение
не добавляется в URL).

Примеры:
  single ; .css        - явно указаны разделитель и расширение
  single ;             - разделитель ";", расширение из обоймы
  single .css          - расширение ".css", разделитель из обоймы
  single               - оба параметра из обойм по умолчанию
  single none .css     - без разделителя, только расширение .css
  single ; none        - разделитель ";", без расширения

🌐 ПРЯМОЙ ЗАПРОС (RAW)
----------------------
raw <URL>
    Выполняет прямой curl-запрос к произвольному URL без добавления test,
    разделителей или расширений. Автоматически использует прокси, если он
    включен. Полезно для проверки чистого эндпоинта или отладки.

Примеры:
  raw http://127.0.0.1:8080/profile
  raw http://127.0.0.1:8080/profile;test.css
  raw http://127.0.0.1:8080/login

📦 УПРАВЛЕНИЕ РАЗДЕЛИТЕЛЯМИ
---------------------------
add <CHAR>            - добавить разделитель в текущую обойму
addmany               - добавить несколько разделителей (ввод через пробел)
remove <CHAR>         - удалить разделитель из обоймы
delimiters            - показать все текущие разделители
clear_delimiters      - очистить обойму разделителей
reset_delimiters      - сбросить к стандартному набору (; . ? %00 %23 %3F %0A %09 %20)

📦 УПРАВЛЕНИЕ РАСШИРЕНИЯМИ
--------------------------
addext <EXT>          - добавить расширение (точка добавляется автоматически)
addextmany            - добавить несколько расширений
remext <EXT>          - удалить расширение
extensions            - показать все текущие расширения
clear_extensions      - очистить обойму расширений
reset_extensions      - сбросить к стандартному набору (.css .js .jpg .ico)

💾 РАБОТА С ОБОЙМАМИ (СОХРАНЕНИЕ НАБОРОВ)
----------------------------------------
Обоймы хранятся в папке "magazines", которая находится в исходной директории
проекта: ~/BB/CORE-SWIGGER/wcd_flask_test/tools/magazines/

При первом запуске папка создаётся автоматически.

Структура:
  ~/BB/CORE-SWIGGER/wcd_flask_test/tools/
  ├── wcd_gun.sh
  ├── wcd_gun.log
  └── magazines/
      ├── my_delims.delimiters
      ├── my_exts.extensions
      └── ...

Файлы разделителей имеют расширение .delimiters
Файлы расширений имеют расширение .extensions

Формат файлов: каждая строка содержит один разделитель или одно расширение.

Пример файла my_delims.delimiters:
  ;
  .
  ?
  %00
  %23

Пример файла my_exts.extensions:
  .css
  .js
  .jpg
  .ico

Команды:
  save_d <имя>        - сохранить текущие разделители в обойму
  load_d <имя>        - загрузить разделители из обоймы (ЗАМЕНЯЕТ текущие)
  list_d              - показать список сохранённых обойм разделителей
  delete_d <имя>      - удалить обойму разделителей

  save_e <имя>        - сохранить текущие расширения в обойму
  load_e <имя>        - загрузить расширения из обоймы (ЗАМЕНЯЕТ текущие)
  list_e              - показать список сохранённых обойм расширений
  delete_e <имя>      - удалить обойму расширений

  magazines           - показать все сохранённые обоймы (разделителей и расширений)

🔥 ПОЛНЫЙ ОБСТРЕЛ
-----------------
fire
    Запускает двухфазный обстрел по ВСЕМ комбинациям текущих разделителей
    и расширений. Перед началом проверяется кука и цель. Ход выполнения
    отображается в реальном времени с подсчётом результатов.

    Фаза 1: Прогрев. Для каждого вектора (разделитель + "test" + расширение)
            отправляется запрос с кукой (или без, если кука не задана).
    Фаза 2: Проверка. Для каждого вектора отправляется запрос БЕЗ куки и
            анализируется заголовок X-Cache-Status.

    Итоги: Количество HIT (уязвимость), BYPASS (защита), MISS (нет в кэше),
            ошибок. Выносится вердикт.

СИСТЕМНЫЕ КОМАНДЫ
------------------
!<команда>
    Выполняет произвольную системную команду bash, не выходя из пулемёта.
    Пример: !curl -i http://127.0.0.1:8080/profile
    Пример: !ls -la magazines/

ДРУГИЕ КОМАНДЫ
--------------
check [URL]           - проверить доступность цели (если URL не указан, текущую)
clear                 - очистить экран и показать баннер
help                  - показать встроенную справку
exit                  - выйти из пулемёта

ЛОГИРОВАНИЕ
-----------
Все действия и ошибки записываются в файл:
  ~/BB/CORE-SWIGGER/wcd_flask_test/tools/wcd_gun.log

Формат лога:
  [дата время] [уровень] сообщение
                детали

Уровни логирования:
  INFO  - обычные действия
  WARN  - предупреждения
  ERROR - ошибки
  VULN  - обнаружена уязвимость
  SHOT  - одиночный выстрел
  FIRE  - полный обстрел
  RAW   - прямой запрос
  START - начало сессии
  STOP  - завершение сессии

ТИПИЧНЫЙ СЦЕНАРИЙ ИСПОЛЬЗОВАНИЯ
-------------------------------
1. Запустить пулемёт: ./wcd_gun.sh
2. На запрос о прокси ответить 'y', если нужно видеть трафик в Burp.
3. Ввести цель: http://127.0.0.1:8080/profile
4. Ввести куку: session=valid
5. Проверить, что цель и кука работают: target и cookie
6. Проверить чистый эндпоинт: raw http://127.0.0.1:8080/profile
7. Настроить разделители (если нужно):
     reset_delimiters
     add %0d%0a
8. Настроить расширения (если нужно):
     reset_extensions
     addext .json
9. (Опционально) Сохранить наборы:
     save_d my_delims
     save_e my_exts
10. Провести одиночный тест: single ; .css
11. Если тест показал HIT, запустить полный обстрел: fire
12. Проанализировать результаты. При обнаружении уязвимости будет выведено
    "ПРОБИТИЕ!" и URL вектора.
13. Проверить лог: !cat wcd_gun.log

СОЗДАНИЕ ОБОЙМ ВРУЧНУЮ
----------------------
Вы можете создать файлы обойм вручную в папке magazines:

  cd ~/BB/CORE-SWIGGER/wcd_flask_test/tools
  mkdir -p magazines
  echo ";" > magazines/my_delims.delimiters
  echo "." >> magazines/my_delims.delimiters
  echo ".css" > magazines/my_exts.extensions
  echo ".js" >> magazines/my_exts.extensions

Затем загрузите их в пулемёте:
  wcd-gun > load_d my_delims
  wcd-gun > load_e my_exts

ПРИМЕЧАНИЯ
----------
- Скрипт использует curl. Убедитесь, что он установлен.
- Для работы прокси через Burp Suite убедитесь, что Burp слушает на
  127.0.0.1:8082 и опция "Support invisible proxying" включена (если нужно).
- Расширения автоматически получают точку в начале, если вы её не указали.
- Обоймы загружаются с полной заменой текущего массива, а не дополнением.
- Логи и обоймы сохраняются в ~/BB/CORE-SWIGGER/wcd_flask_test/tools/
  независимо от того, откуда запущен скрипт (даже из /tmp).
